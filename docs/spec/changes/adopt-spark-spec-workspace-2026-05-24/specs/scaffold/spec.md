## MODIFIED Requirements

### Requirement: Minimal AI-Ready Scaffold (nextjs reference)

The `create-spark` initializer SHALL generate a minimal Next.js 15 + TypeScript application when the user selects the `nextjs` template. The generated app MUST be preloaded with the spark workflow artifacts (`AGENTS.md`, `CLAUDE.md`, a `docs/spark/` spec workspace, `.claude/skills/`, `.codex/skills/`) and nothing else. The generated app MUST NOT include auth, database, UI component library, payments, email, or AI SDK integrations by default; those capabilities are added later via the pack registry. The generated tree MUST NOT contain a `.ai/` directory.

#### Scenario: Fresh init with nextjs and no preset

- **WHEN** the user runs `bunx create-spark my-app --template nextjs` and declines any preset
- **THEN** a directory `my-app/` is created containing a runnable Next.js 15 app
- **AND** `my-app/AGENTS.md`, `my-app/CLAUDE.md`, `my-app/docs/spark/`, `my-app/.claude/skills/`, and `my-app/.codex/skills/` exist
- **AND** `my-app/docs/spark/project.md`, `my-app/docs/spark/design.md`, and `my-app/docs/spark/AGENTS.md` exist
- **AND** no `my-app/.ai/` directory exists
- **AND** `bun dev` starts the dev server successfully without prompting for environment variables
- **AND** no auth, db, UI lib, payments, email, or AI SDK code is present in the generated tree

#### Scenario: Init with a preset

- **WHEN** the user runs `bunx create-spark my-app --template nextjs --preset saas-classic`
- **THEN** the base scaffold is generated as in the no-preset case
- **AND** the CLI subsequently invokes `spark preset saas-classic` against the new directory
- **AND** the packs declared by that preset are installed before the command returns

### Requirement: Scaffold Carries Canonical Skills

The base scaffold copied by `create-spark` SHALL include the canonical set of spark-workflow skills under `.claude/skills/`, and a mirrored copy under `.codex/skills/` generated by the project-wide skill mirror. Pack-shipped skills are added later when packs install, not at scaffold time.

#### Scenario: Skills are present after init

- **WHEN** the scaffold finishes
- **THEN** `.claude/skills/` contains at minimum `start`, `build-loop`, `capture-feedback`, `mvp-grill`, `mvp-spec`, `architecture-cutline`, `ux-theme`, `mvp-board`, `board-review`, `parallel-execution`, `implementation-brief`, `execute-task`, `code-review`, `qa-verify`, `sync-board`, `next-task`, `risk-check`, `pack-resolve`, `pack-add`, and `new-pack`
- **AND** every skill present in `.claude/skills/` has a corresponding entry in `.codex/skills/`
